Background

- Non-Software Engineering exists of a lot of repetitive work

- Work exist of smaller sub-problems that are easy to solve

- Yet, there is no place to collaborate on these problems

Idea

- Sndbxx is much like Codepen/JSFiddle

- Enabling Software Tools for Non-Software Problems

- Providing all required backend and libraries


Business Model

- Using and editing is free

- Revenue comes from private development for proprietary tools

- Or by connecting Engineers to the vendors they need using this tool

- Alternatively, revenues may come from ads, but NEVER in an annoying way

Approach

We want to start small and specific. We know Structural Engineering and we can build a product that Structual Engineers will love. But we want to build our platform in a generic way so that it is easy to add libraries and support for other fields later on. Some thought on the fields of play that we think can use a tool like ours:


- Structural Engineering
- Control Engineering
- Charts and Statistics (yes, quite some competition, but no one with a similar business model)
- Chemical Engineering
- Electrical Engineering
- Non-Engineering fields
...
